On a trouve le disque /dev/sda.

On efface avec gparted les partitions existantes: 

root@ubuntu:/home/ubuntu# gparted /dev/sda
======================
libparted : 2.3
======================

Dans l'outil graphique, on demonte ou desactive ou efface les partitions existantes et on valide.

Le disque est vide sans partitions

On cree une nouvelle partition:

  • New size : 500
  • File system: ext2
  • Label : boot

On valide.

Creation du volume physique LVM sur le reste de l'espace disque.

On cree une nouvelle partition:

  • New size : espace par defaut restant
  • File system: lvm2 pv
  • Label :

On valide.

On quitte gparted.

Dans le terminal on va cree le volume group et les partitions logiques. 

root@ubuntu:/home/ubuntu# pvs
  PV         VG   Fmt  Attr PSize   PFree  
  /dev/sda2       lvm2 a--  232,34g 232,34g

root@ubuntu:/home/ubuntu# vgcreate vg_messina /dev/sda2
  Volume group "vg_messina" successfully created

root@ubuntu:/home/ubuntu# vgs
  VG         #PV #LV #SN Attr   VSize   VFree  
  vg_messina   1   0   0 wz--n- 232,34g 232,34g

root@ubuntu:/home/ubuntu# lvcreate -L+10G -n root vg_messina
  Logical volume "root" created
root@ubuntu:/home/ubuntu# lvcreate -L+10G -n tmp vg_messina
  Logical volume "tmp" created
root@ubuntu:/home/ubuntu# lvcreate -L+100G -n home vg_messina
  Logical volume "home" created

Pour information, si il y a un besoin d'extension d'une partition montee, voici les commandes: 

root@ubuntu:/home/ubuntu# lvextend -L+5G /dev/vg_messina/tmp 

  Extending logical volume tmp to 15,00 GiB

  Logical volume tmp successfully resized

root@ubuntu:/home/ubuntu# resize2fs /dev/vg_messina/tmp 

resize2fs 1.42.9 (4-Feb-2014)

resize2fs: Numéro magique invalide dans le super-bloc lors de la tentative d'ouverture de /dev/vg_messina/tmp

Impossible de trouver un superbloc de système de fichiers valide.

root@ubuntu:/home/ubuntu# lvs

  LV   VG         Attr      LSize   Pool Origin Data%  Move Log Copy%  Convert

Les partitions sont pretes. On passe a l'installation systeme.

Dans le choix du partitionnement disque, choisir Autre chose.

utiliser comme formater point de montage
/dev/mapper/vg_messina-home ext4 x /home
/dev/mapper/vg_messina-root ext4 x /
/dev/mapper/vg_messina-swap swap
/dev/mapper/vg_messina-tmp ext4 x /tmp
/dev/sda1 ext2 x /boot
